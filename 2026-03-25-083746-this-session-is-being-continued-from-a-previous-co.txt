     4.1 Backend: Direct Query Endpoint

     File: contextual_cache/api/routes.py

     class DirectQueryRequest(BaseModel):
         query: str = Field(..., min_length=1, max_length=10000)

     class DirectQueryResponse(BaseModel):
         response: str
         latency_ms: float
         input_tokens: int = 0
         output_tokens: int = 0
         total_tokens: int = 0

     @router.post("/api/query-direct", response_model=DirectQueryResponse)
     async def direct_query_endpoint(req: DirectQueryRequest):
         """Bypass cache, call LLM directly. For comparison demos."""
         cm = _get_cm()
         t0 = time.monotonic()
         llm_resp = await cm.llm_provider.generate(req.query)
         latency = (time.monotonic() - t0) * 1000
         return DirectQueryResponse(
             response=llm_resp.text,
             latency_ms=round(latency, 2),
             input_tokens=llm_resp.input_tokens,
             output_tokens=llm_resp.output_tokens,
             total_tokens=llm_resp.input_tokens + llm_resp.output_tokens,
         )

     Phase 5: How Things Work (For Demonstration)

     These are explanations to be aware of for the demo, not code changes:

     Thompson Sampling

     - Each of 10 arms represents a candidate similarity threshold [0.65..0.95]
     - Arms maintain Beta(Оұ, ОІ) posteriors updated by feedback
     - On each query, the bandit samples from all arms and uses the best
     - Over time, arms with higher precision get higher Оұ вҶ’ selected more often
     - ADWIN drift detection resets posteriors if query distribution shifts

     Adaptive Conformal Thresholds

     - Each cached entry has its own threshold П„_i
     - Correct hits push П„_i lower (more permissive); incorrect hits push it
     higher
     - Uses (1-Оө)-quantile of nonconformity scores as the threshold
     - Guarantees P(false positive) вүӨ Оө (currently 0.05) per entry

     Redis (Future Work)

     - Replace _exact_store dict with Redis for Tier 1 (distributed exact-hash)
     - Redis Cluster for multi-node sharding
     - Redis Streams for gossip protocol alternative
     - Redis TimeSeries for metrics persistence
     - redis_enabled: bool = False already in config, ready for integration

в—Ҹ Here's how it works in your system вҖ” two layers working together:

  Thompson Sampling Bandit (Global Prior)

  The problem it solves: When a new entry enters the cache, what similarity
  threshold should it use? Too low = false positives (wrong cached answers
  served). Too high = wasted cache hits (correct matches rejected).

  How it works:

  1. Arms = candidate thresholds. 10 arms linearly spaced from 0.65 to 0.95:
  [0.65, 0.683, 0.717, 0.75, 0.783, 0.817, 0.85, 0.883, 0.917, 0.95]
  2. Each arm has a Beta(Оұ, ОІ) distribution. Оұ counts successes (correct cache
  hits), ОІ counts failures (false positives). Initially all arms start at
  Beta(1,1) = uniform = "no opinion."
  3. On each query, the bandit samples from each arm's Beta distribution and
  picks the arm with the highest sample. This is the explore/exploit tradeoff вҖ”
  arms with more successes have higher Оұ so they tend to produce higher samples,
   but randomness ensures underexplored arms still get tried.
  4. After a cache hit, feedback flows in:
    - Correct hit вҶ’ reward = 1.0 вҶ’ Оұ increases for that arm (arm looks better)
    - Incorrect hit вҶ’ reward = 0.0 вҶ’ ОІ increases (arm looks worse)
    - Uncertain вҶ’ skipped entirely (no noise injection)
  5. Over time, the arm at the "sweet spot" threshold accumulates the most Оұ
  relative to ОІ. For your dataset with paraphrase-MiniLM-L6-v2, this will likely
   converge around 0.75-0.80.
  6. Drift detection (ADWIN): If the query distribution shifts (e.g., users
  start asking very different topics), the reward stream changes. ADWIN detects
  this statistical shift and resets all posteriors to Beta(2,2) вҖ” a mild reset
  that says "start learning again but with slight optimism."

  Per-Entry Conformal Thresholds (Entry-Specific)

  The problem it solves: A single global threshold is crude. "What is Python?"
  might need П„=0.72 (lots of valid paraphrases), while "What is the capital of
  France?" might need П„=0.90 (very specific, slight differences change the
  answer).

  How it works:

  1. New entries start at the default_threshold (0.75, informed by the bandit's
  prior).
  2. Feedback accumulates per entry as nonconformity scores:
    - Correct hit at similarity 0.85 вҶ’ nonconformity = 1 - 0.85 = 0.15 (low =
  good)
    - Incorrect hit at similarity 0.78 вҶ’ nonconformity = max(1 - 0.78 + 0.1, 
  0.5) = 0.5 (high = penalty)
  3. After 10+ feedback signals (the min_cal threshold), the conformal method
  kicks in:
    - Sort all nonconformity scores
    - Take the (1-Оө)-quantile (with Оө=0.05, this is the 95th percentile)
    - Threshold = 1 - that quantile value
  4. The guarantee: By construction, this gives P(false positive) вүӨ Оө = 5% per
  entry. This is a formal statistical guarantee from conformal prediction theory
   вҖ” not a heuristic.
  5. Sliding window (max 200 scores) ensures the threshold adapts if the entry's
   match pattern changes over time.

  How They Connect

  New entry arrives вҶ’ uses bandit's best threshold as default
                           вҶ“
  Queries come in вҶ’ conformal store collects feedback per entry
                           вҶ“
  After 10+ feedbacks вҶ’ conformal threshold takes over (entry-specific)
                           вҶ“
  Feedback also flows to bandit вҶ’ bandit learns better default for NEXT new
  entry
                           вҶ“
  If distribution shifts вҶ’ ADWIN resets bandit вҶ’ re-learns from scratch

  The bandit is the exploration engine for the system-wide prior. Conformal
  thresholds are the per-entry precision engine. Together, new entries start
  with a good guess (bandit) and quickly adapt to their own optimal threshold
  (conformal). This is why the system should outperform static-threshold systems
   like GPTCache on precision вҖ” which your benchmark already shows (44.44% vs
  40%).
